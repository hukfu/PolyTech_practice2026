# Почему сайт перенесён в корень (root), а не лежит в папке `site`

## Причина

GitHub Pages по умолчанию ищет файлы для публикации только в двух местах:

1. В **корне репозитория** (`/`)
2. В папке **`/docs`**

Папку с любым другим именем (например, `site`) GitHub Pages **игнорирует** — это ограничение платформы, а не ошибка.

## Что было сделано

Исходно все файлы сайта лежали в `site/`
Но для корректной работы GitHub Pages все файлы были **перенесены в корень репозитория**
