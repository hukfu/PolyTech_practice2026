# Почему сайт перенесён в корень (root), а не лежит в папке `site`

## Причина

GitHub Pages по умолчанию ищет файлы для публикации только в двух местах:

1. В **корне репозитория** (`/`)
2. В папке **`/docs`**

Папку с любым другим именем (например, `site`) GitHub Pages **игнорирует** — это ограничение платформы, а не ошибка.

## Что было сделано

Исходно все файлы сайта лежали в `site/`:
site/
├── index.html
├── about.html
├── team.html
├── journal.html
├── resources.html
└── images/

Для корректной работы GitHub Pages все файлы были **перенесены в корень репозитория**:
/
├── index.html
├── about.html
├── team.html
├── journal.html
├── resources.html
└── images/


## Где теперь искать файлы сайта

- **Живой сайт**: `https://[логин].github.io/[репозиторий]/`
- **Исходники в корне**: там же, в репозитории
- **Резервная копия в папке `site`**: сохранена для истории, но не используется GitHub Pages


**Выбран простой и надёжный путь** — файлы в корне, сайт работает гарантированно.
